PHASE 7: MULTI-MODAL ARCHITECTURE WITH 4 VALIDATION LAYERS
===========================================================

FILE: /e/icu_project/src/models/ensemble_predictor.py (553 lines)

COMPONENTS IMPLEMENTED:

1. DualModelEnsemblePredictor (Main Class)
   ├─ Dual-Path Architecture:
   │  ├─ Deep Learning Path: PyTorch Transformer
   │  │  ├─ Multi-head attention (8 heads)
   │  │  ├─ Positional encoding for temporal data
   │  │  ├─ MC Dropout for uncertainty (5 stochastic passes)
   │  │  └─ Output: (mortality_prob, confidence, uncertainty)
   │  │
   │  └─ Machine Learning Path: Random Forest Ensemble
   │     ├─ Random Forest (tuned: 0.9032 AUC)
   │     ├─ Input: 120 engineered features (aggregated vitals + labs)
   │     ├─ RobustScaler normalization
   │     └─ Output: (mortality_prob, confidence)
   │
   ├─ Model Agreement Analyzer
   │  ├─ Calculates DL-ML concordance (0-1 scale)
   │  ├─ Threshold-based flagging (>0.15 difference = uncertain)
   │  ├─ Analyzes disagreement patterns
   │  └─ Identifies when to distrust predictions
   │
   └─ Comprehensive Confidence System
      ├─ Base confidence: 60% ML + 40% DL weighted
      ├─ Validation penalties: -0.25 per failed check
      ├─ Range: 0-1 (lower = requires clinical review)
      └─ Automatic fallback if one model unavailable

2. ValidationLayer1_Concordance
   ├─ Purpose: Detect when DL and ML strongly disagree
   ├─ Mechanism: Normalized absolute difference check
   ├─ Threshold: |p_dl - p_ml| > 0.15 flags as uncertain
   ├─ Severity: MEDIUM
   ├─ Confidence Penalty: 0.2
   └─ Recommendation: Clinical review recommended

3. ValidationLayer2_ClinicalRules
   ├─ Purpose: Validate against medical knowledge
   ├─ Rule 1: "Normal vitals → risk <20%"
   ├─ Rule 2: "Critical vitals → risk >50%"
   ├─ Extracts vital signs from temporal data
   ├─ Configurable vital ranges from INDIAN_HOSPITAL_CONFIG
   ├─ Severity: HIGH (violating clinical knowledge = big red flag)
   └─ Confidence Penalty: 0.25

4. ValidationLayer3_CohortConsistency
   ├─ Purpose: Compare with similar patient outcomes
   ├─ Mechanism: Find k=10 nearest neighbors in historical database
   ├─ Calculates cohort mortality rate baseline
   ├─ Flags if prediction deviates >0.25 from cohort
   ├─ Severity: MEDIUM (outlier detection)
   └─ Confidence Penalty: 0.15

4. ValidationLayer4_TrajectoryConsistency
   ├─ Purpose: Detect sudden changes in patient risk
   ├─ Mechanism: Calculate risk trajectory over timesteps
   ├─ Detects jumps: max_diff > 0.25 triggers flag
   ├─ Detects zigzag patterns (inconsistent trends)
   ├─ Severity: MEDIUM
   └─ Confidence Penalty: 0.15

PREDICTION OUTPUT STRUCTURE:
{
    'mortality_risk': 0.35,          # Main prediction (0-1)
    'mortality_percent': '35.0%',    # Formatted for display
    'risk_class': 'MEDIUM',          # LOW/MEDIUM/HIGH/CRITICAL
    'confidence': 0.82,              # Post-validation confidence
    'dl_prediction': 0.38,           # Transformer output
    'ml_prediction': 0.33,           # Random Forest output
    'dl_confidence': 0.85,           # DL confidence before penalties
    'ml_confidence': 0.80,           # ML confidence before penalties
    'agreement_score': 0.95,         # DL-ML concordance (0-1)
    'agreement_analysis': {...},     # Disagreement details
    'validation_results': {
        'layer1_concordance': {...},
        'layer2_clinical_rules': {...},
        'layer3_cohort': {...},
        'layer4_trajectory': {...}
    },
    'validation_flags_count': 0,     # How many checks failed
    'validation_status': 'PASSED',   # PASSED or FLAGGED_FOR_REVIEW
    'explanation': 'Both models agree...'  # Human-readable summary
}
