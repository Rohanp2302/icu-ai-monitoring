PHASE 7: MULTI-MODAL PREDICTION
================================

Python Usage:
-----------
from src.models.ensemble_predictor import DualModelEnsemblePredictor
import numpy as np

# Initialize
predictor = DualModelEnsemblePredictor(
    ml_model_path='results/dl_models/best_model.pkl',
    ml_scaler_path='results/dl_models/scaler.pkl'
)

# Prepare data
x_temporal = np.random.randn(24, 42)  # 24 hours × 42 temporal features
x_static = np.random.randn(120)       # 120 engineered features

# Get prediction
result = predictor.predict(x_temporal, x_static)

# Output:
print(result['mortality_risk'])           # 0.35 (35% mortality risk)
print(result['risk_class'])               # 'MEDIUM'
print(result['confidence'])               # 0.82 (confidence score)
print(result['validation_status'])        # 'PASSED' or 'FLAGGED_FOR_REVIEW'
print(result['validation_results'])       # Details from all 4 layers

PHASE 8: MEDICINE TRACKING
===========================

Python Usage:
-----------
from src.medicine.medicine_tracker import MedicineTracker

# Initialize

# Add medications

    med_name='Noradrenaline',
    dose='0.15 mcg/kg/min',
    frequency='Continuous IV',
    reason='Blood pressure support'
)

tracker.add_medication(
    med_name='Propofol',
    dose='1.5 mg/kg/h',
    frequency='Continuous IV',
    reason='Sedation'
)

# Check interactions (automatic)
# Returns: CRITICAL interaction detected (Noradrenaline + Propofol)

# Check all interactions
interactions = tracker.check_all_interactions()
print(interactions['total'])              # Total interactions
print(interactions['critical'])           # Critical ones
print(interactions['safe'])               # Safe to proceed?

# Predict adverse events
vitals = {
    'heart_rate': 125,
    'blood_pressure_systolic': 130,
    'oxygen_saturation': 95
}

events = tracker.predict_adverse_events(vitals)
# Returns: List of predicted adverse events with severity

# Generate family summary
summary = tracker.generate_summary(language='en')
print(summary)  # Family-friendly medication list + alerts

UNDERSTANDING VALIDATION LAYERS
================================

Layer 1: Concordance Check
├─ What it checks: Do DL and ML agree?
├─ Threshold: Difference > 0.15 = flag
├─ Meaning: Models disagree = uncertainty
└─ Action: Review prediction manually

Layer 2: Clinical Rules Check
├─ What it checks: Does prediction violate medical knowledge?
├─ Rules: "Normal vitals → <20% risk", "Critical vitals → >50% risk"
├─ Meaning: Prediction doesn't make clinical sense
└─ Action: Investigate inconsistency

Layer 3: Cohort Consistency Check
├─ What it checks: Does prediction match similar patients?
├─ Mechanism: Compare with 10 similar patients' outcomes
├─ Meaning: Prediction is outlier for this patient type
└─ Action: Consider whether patient is truly unique

Layer 4: Trajectory Consistency Check
├─ What it checks: Is patient's risk changing logically?
├─ Detects: Sudden jumps (>0.25 change), zigzag patterns
├─ Meaning: Risk prediction is unstable
└─ Action: Wait for stabilization or increase monitoring

CONFIDENCE SCORE INTERPRETATION
================================

Confidence = Base_Confidence - Validation_Penalties

High Confidence (0.8-1.0):
├─ Both models agree
├─ All validation layers passed
├─ Prediction clinically plausible
├─ Risk score reliable
└─ Action: Use for clinical decisions

Medium Confidence (0.6-0.8):
├─ Some validation flags
├─ Minor model disagreement acceptable
├─ Consider with other factors
└─ Action: Use with clinical judgment

Low Confidence (0.3-0.6):
├─ Multiple validation failures
├─ Significant model disagreement
├─ Prediction marked uncertain
└─ Action: Clinical review recommended

Very Low Confidence (<0.3):
├─ System recommends NOT using prediction alone
├─ Extensive validation failures
├─ Requires full clinical assessment
└─ Action: Do NOT use without clinical review

SUPPORT & EXTENSIONS
====================

To add new drug interactions:
- Edit CRITICAL_DRUG_INTERACTIONS in medicine_tracker.py
- Format: ('Drug1', 'Drug2'): {'severity': '...', 'mechanism': '...', ...}

To adjust vital ranges:
- Edit INDIAN_HOSPITAL_CONFIG in indian_hospital_config.py
- Can customize per hospital/region

To add new languages:
- Add language code to FamilyExplainerEngine class
- Translate all explanation strings

To change validation thresholds:
- Edit ValidationLayer classes in ensemble_predictor.py
- Adjust 'threshold' parameters

To update model:
- Replace results/dl_models/best_model.pkl with new trained model
- Ensure same 120 features expected
